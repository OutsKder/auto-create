## 2. 数据库核心实体设计 (Database Schema)

作为首席架构师，我必须强调**财务数据的精度**和**整型数据的边界**。虽然需求简单，但底层模型必须规范，防止未来的累加错误。

### 2.1 表结构变更 (`assets` / `items`)

建议对现有的 `items` 表进行 Alter Table 操作。为了兼容历史旧数据，所有新增字段允许为 `NULL` 或设置合理的默认值。

```sql
-- 核心表：assets (原 items)
ALTER TABLE assets 
ADD COLUMN unit_price DECIMAL(10, 2) DEFAULT 0.00 COMMENT '单价，精确到分',
ADD COLUMN stock_quantity INT DEFAULT 0 COMMENT '当前库存数量',
ADD COLUMN updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '最后更新时间';

-- 索引优化 (考虑未来按价格或库存范围查询)
CREATE INDEX idx_stock_alert ON assets(stock_quantity);
```

### 2.2 设计要点说明
1.  **Decimal vs Float**: `price` 字段严禁使用 `FLOAT` 或 `DOUBLE`。浮点数运算存在精度丢失风险（如 `0.1 + 0.2 != 0.3`），财务相关字段必须使用 `DECIMAL(M, D)` 类型，这里定为 `10, 2`。
2.  **非负约束**: `stock` 应当在数据库层面通过 `CHECK` 约束（若支持）或应用层校验保证 `>= 0`，避免库存变成负数。
3.  **向后兼容**: 默认值设为 0，确保旧数据导入后不会引发除零错误或显示异常。

## 3. API 接口设计思路 (API Design)

遵循 RESTful 规范，确保接口的幂等性和安全性。

### 3.1 物品创建/更新接口
*   **Endpoint**: `POST /api/v1/assets` (新建) 或 `PUT /api/v1/assets/{id}` (更新)
*   **Content-Type**: `application/json`
*   **请求体 (Payload)**:
    ```json
    {
      "name": "笔记本电脑 A 型",
      "description": "运维专用",
      "price": 8999.00,  // 验证：正数，两位小数
      "stock": 15        // 验证：非负整数
    }
    ```
*   **响应体 (Response)**:
    ```json
    {
      "code": 200,
      "data": {
        "id": 101,
        "status": "active",
        "low_stock_alert": false // 服务端直接返回告警状态，减轻前端计算压力
      }
    }
    ```

### 3.2 列表查询接口
*   **Endpoint**: `GET /api/v1/assets`
*   **Query Params**: `page=1&size=20`
*   **扩展**: 建议在列表返回中增加 `is_urgent` 字段，值为布尔型。由后端计算 `(stock < 5)` 并注入，前端只需关注样式，降低前后端耦合。

## 4. 前端业务逻辑实现 (Frontend Logic)

针对 Bootstrap 环境和即时性需求，采取轻量级实现。

### 4.1 表单组件升级
在现有的 `Add Item Form` 中插入如下字段组：

```html
<!-- 单价输入框 -->
<div class="mb-3">
    <label class="form-label">单价 (CNY)</label>
    <input type="number" step="0.01" min="0.01" class="form-control" name="price" required>
</div>

<!-- 库存输入框 -->
<div class="mb-3">
    <label class="form-label">初始库存</label>
    <input type="number" step="1" min="0" class="form-control" name="stock" required>
</div>
```
*   **前端校验**: 利用 HTML5 `type="number"` 和 `step` 属性限制输入格式，减少无效请求。

### 4.2 列表卡片渲染逻辑
根据需求，在 Title 旁动态渲染 Badge。建议封装一个简单的 Helper 函数或 Vue/React 计算属性。

**伪代码逻辑**:
```javascript
// 数据对象 item
function renderBadge(item) {
    // 需求核心逻辑：库存 < 5 显示警告
    if (item.stock < 5) {
        return '<span class="badge bg-danger ms-2">库存告急</span>';
    }
    return ''; // 不显示或显示绿色正常态，视美观度定
}
```
