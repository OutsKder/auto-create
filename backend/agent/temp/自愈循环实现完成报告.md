## 🎯 核心功能

### 1. ErrorType - 错误分类 (14 种)

```
✓ SyntaxError       - Python 语法错误
✓ ImportError       - 导入和依赖问题
✓ RuntimeError      - 运行时错误
✓ AttributeError    - 属性访问错误
✓ TypeError         - 类型错误
✓ ValueError        - 值错误
✓ KeyError          - 键错误
✓ IndexError        - 索引错误
✓ AssertionFailed   - 测试断言失败
✓ Timeout           - 执行超时
✓ PermissionError   - 权限问题
✓ NetworkError      - 网络问题
✓ EnvironmentError  - 环境配置问题
✓ Unrecoverable     - 不可恢复错误
```

### 2. TriageAgent - 失败诊断

**职责：** 分析失败日志，生成诊断报告

**功能：**

- 📝 日志解析 - 从 stdout/stderr 提取错误信息
- 🏷️ 错误分类 - 基于关键字识别错误类型
- 🔍 原因分析 - 根据错误类型提供根本原因分析
- 💡 修复建议 - 生成给 CodeGen 的具体修复指导
- 📊 置信度计算 - 评估分析准确性 (0-1)

**输入/输出：**

```python
# 输入
{
    "sandbox_result": SandboxResult,     # 测试执行结果
    "code_changes": str,                  # 本轮生成的代码
    "previous_failures": [FailureAnalysis],  # 历史失败记录
}

# 输出
{
    "failure_analysis": FailureAnalysis,   # 详细失败分析
    "fix_suggestion": str,                 # 修复建议
    "retry_decision": bool,                # 是否可重试
    "reason": str,                         # 决策原因
}
```

### 3. RetryManager - 重试管理

**职责：** 管理重试循环，防止陷阱

**防护机制：**

- ✓ 最大重试次数限制 (默认 3 次)
- ✓ 循环检测 (相同错误 2 次即停止)
- ✓ 错误类型检查 (UNRECOVERABLE 无法恢复)
- ✓ 状态追踪 (记录重试历史)

**核心方法：**

```python
should_continue(failure) → bool      # 判断是否继续重试
record_failure(failure) → None       # 记录失败
get_status() → dict                  # 获取当前状态
reset() → None                       # 重置管理器
```

### 4. SelfHealingCoordinator - 自愈协调器

**职责：** 协调整个自愈循环的执行流程

**执行流程：**

```
迭代 N
  ↓
[1] CodeGen → 生成/修复代码
  ↓
[2] WorkspaceManager → 创建隔离工作区
  ↓
[3] Patcher → 应用代码补丁
  ↓
[4] SDETAgent → 生成测试套件
  ↓
[5] Runner → 在 Docker 中执行测试
  ↓
  ✅ 测试通过? → 完成，返回成功报告
  ❌ 测试失败 ↓

[6] TriageAgent → 分析失败原因
  ↓
[7] RetryManager → 判断是否继续迭代
  ↓
  ✓ 应该继续 → 反馈给 CodeGen, 迭代 N+1
  ✗ 应该停止 → 返回失败报告
```

**输入/输出：**

```python
# 输入
{
    "requirement_raw": str,        # 原始需求
    "codebase": {
        "repo_path": str,          # 代码库路径
    }
}

# 输出
SelfHealingReport (包含):
  - success: bool                  # 最终是否成功
  - iterations: int                # 迭代次数
  - final_code: Patch[]           # 最终代码补丁
  - test_results: dict            # 测试结果
  - failure_history: FailureAnalysis[]  # 失败历史
  - total_time: float             # 总耗时
  - iterations_log: SelfHealingIteration[]  # 迭代日志
```

## 💡 使用示例

```python
from backend.agent import SelfHealingCoordinator

# 初始化协调器
coordinator = SelfHealingCoordinator(max_retries=3, use_docker=True)

# 执行自愈循环
report = coordinator.execute_with_self_healing({
    "requirement_raw": "实现一个 REST API",
    "codebase": {
        "repo_path": "/path/to/project",
    }
})

# 查看结果
if report.success:
    print(f"✅ 成功！耗时 {report.total_time:.1f}s，迭代 {report.iterations} 次")
else:
    print(f"❌ 失败。最后错误: {report.final_failure.error_type}")
    print(f"修复建议: {report.final_failure.suggestion}")
```
