## 1. 整体流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                        需求分析 + 设计                            │
│  (RequirementAnalyst, TechArchitect) → 需求结构化、代码方案        │
└────────────────────────┬────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────────┐
│                    第N次迭代循环                                  │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 1. CodeGen: 生成/修复代码补丁                            │    │
│  │    输入: 需求 + (失败原因分析 if N>1)                   │    │
│  │    输出: Patch[] (代码修改)                             │    │
│  └────────────┬────────────────────────────────────────────┘    │
│               ↓                                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 2. WorkspaceManager: 创建隔离工作区                      │    │
│  │    输入: 代码补丁                                       │    │
│  │    输出: 工作区路径 + 补丁应用状态                       │    │
│  └────────────┬────────────────────────────────────────────┘    │
│               ↓                                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 3. SDETAgent: 生成测试套件                              │    │
│  │    输入: 工作区 + 代码修改                              │    │
│  │    输出: TestBundle (test_code, runner_commands)       │    │
│  └────────────┬────────────────────────────────────────────┘    │
│               ↓                                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 4. Runner: 在 Docker 中执行测试                         │    │
│  │    输入: 测试文件 + 命令                                 │    │
│  │    输出: SandboxResult (passed, exit_code, logs)       │    │
│  └────────────┬────────────────────────────────────────────┘    │
│               ↓                                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 5. 判断结果                                              │    │
│  │    ✅ passed=True  → 完成，返回成功结果                  │    │
│  │    ❌ passed=False → 转向失败分析                       │    │
│  └────────────┬────────────────────────────────────────────┘    │
│               ↓ ❌                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 6. TriageAgent: 分析失败原因                            │    │
│  │    输入: SandboxResult (logs, stderr)                  │    │
│  │    输出: FailureAnalysis                               │    │
│  │    - error_type: SyntaxError | RuntimeError | ...      │    │
│  │    - root_cause: 详细根源分析                           │    │
│  │    - suggestion: CodeGen 修复建议                      │    │
│  └────────────┬────────────────────────────────────────────┘    │
│               ↓                                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 7. RetryManager: 决策是否继续迭代                        │    │
│  │    检查:                                                │    │
│  │    - N < MAX_RETRIES (比如 3 次)                       │    │
│  │    - error_type != UNRECOVERABLE                       │    │
│  │    - 是否已修复相同问题                                 │    │
│  └────────────┬────────────────────────────────────────────┘    │
│               ↓                                                   │
│       是否继续迭代？                                              │
│    ✅ 是 → 回到步骤 1 (N:=N+1)                                │
│    ❌ 否 → 返回最终失败报告                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 核心组件设计

### 2.1 FailureAnalysis 数据模型

```python
class ErrorType(Enum):
    """错误分类"""
    SYNTAX_ERROR = "syntax_error"           # Python 语法错误
    IMPORT_ERROR = "import_error"           # 导入问题
    RUNTIME_ERROR = "runtime_error"         # 运行时错误
    ASSERTION_FAILED = "assertion_failed"   # 测试断言失败
    TIMEOUT = "timeout"                     # 超时
    PERMISSION_ERROR = "permission_error"   # 权限问题
    NETWORK_ERROR = "network_error"         # 网络问题
    ENVIRONMENT_ERROR = "environment_error" # 环境配置问题
    UNRECOVERABLE = "unrecoverable"         # 无法恢复

class FailureAnalysis(BaseModel):
    """失败分析结果"""
    error_type: ErrorType
    error_line: int                  # 错误行号
    error_message: str               # 错误信息
    code_snippet: str                # 相关代码片段
    root_cause: str                  # 根本原因分析
    suggestion: str                  # CodeGen 修复建议
    confidence: float                # 分析置信度 (0-1)
    context: dict                    # 上下文信息
    remediation_hint: str            # 修复提示
```

### 2.2 TriageAgent (失败诊断)

**职责：** 分析 SandboxResult，识别错误类型和原因

```python
class TriageAgent(BaseAgent):
    """失败分析代理"""

    def execute(self, context: dict) -> dict:
        """
        输入 context:
        {
            "sandbox_result": SandboxResult,
            "code_changes": str,  # 本轮生成的代码改动
            "previous_failures": [FailureAnalysis],  # 之前的失败记录
        }

        输出:
        {
            "failure_analysis": FailureAnalysis,
            "fix_suggestion": str,
            "retry_decision": bool,
            "reason": str,
        }
        """
        logs = context["sandbox_result"].logs

        # 1. 解析日志，提取错误
        error_info = self._parse_logs(logs)

        # 2. 识别错误类型
        error_type = self._classify_error(error_info)

        # 3. 分析根本原因
        root_cause = self._analyze_root_cause(error_info, code_changes)

        # 4. 生成修复建议给 CodeGen
        suggestion = self._generate_suggestion(error_type, root_cause)

        return {
            "failure_analysis": FailureAnalysis(...),
            "fix_suggestion": suggestion,
            "retry_decision": self._should_retry(error_type),
        }
```

### 2.3 RetryManager (重试管理)

**职责：** 管理迭代循环，防止无限循环

```python
class RetryManager:
    """重试管理器"""

    def __init__(self, max_retries: int = 3):
        self.max_retries = max_retries
        self.retry_count = 0
        self.failure_history = []  # 历史失败记录

    def should_continue(self, failure: FailureAnalysis) -> bool:
        """
        决策是否继续重试

        检查项：
        1. 重试次数是否超限
        2. 错误类型是否可恢复
        3. 是否在重复相同错误（检测循环）
        """
        if self.retry_count >= self.max_retries:
            return False

        if failure.error_type == ErrorType.UNRECOVERABLE:
            return False

        # 检测循环：同一错误出现 2 次则放弃
        if self._is_repeating_error(failure):
            return False

        return True

    def record_failure(self, failure: FailureAnalysis):
        """记录失败，用于循环检测"""
        self.failure_history.append(failure)
        self.retry_count += 1

    def _is_repeating_error(self, failure: FailureAnalysis) -> bool:
        """检测是否在重复相同错误"""
        if len(self.failure_history) < 2:
            return False

        last_error = self.failure_history[-1]
        return (failure.error_type == last_error.error_type and
                failure.error_line == last_error.error_line)
```

### 2.4 SelfHealingCoordinator (自愈协调器)

**职责：** 协调整个自愈循环流程

```python
class SelfHealingCoordinator:
    """自愈循环协调器"""

    def __init__(self):
        self.codegen = CodeGeneratorAgent()
        self.sdet = SDETAgent()
        self.runner = Runner(use_docker=True)
        self.triage = TriageAgent()
        self.retry_manager = RetryManager(max_retries=3)
        self.workspace_manager = WorkspaceManager()

    def execute_with_self_healing(self, context: dict) -> dict:
        """
        执行代码生成 + 测试 + 自愈循环

        输入 context:
        {
            "requirement_raw": str,
            "codebase": {...}
        }

        输出:
        {
            "success": bool,
            "iterations": int,  # 迭代次数
            "final_code": Patch[],
            "test_results": SandboxResult,
            "failure_history": [FailureAnalysis],
        }
        """

        iteration = 0
        feedback_for_codegen = None

        while True:
            iteration += 1
            print(f"\n{'='*60}")
            print(f"第 {iteration} 次迭代")
            print(f"{'='*60}")

            # 1. CodeGen：生成或修复代码
            codegen_input = {
                "requirement": context["requirement_raw"],
                "codebase": context["codebase"],
                "failure_feedback": feedback_for_codegen,  # ◄ 失败反馈
            }
            codegen_result = self.codegen.execute(codegen_input)
            patches = codegen_result.get("patches", [])

            # 2. WorkspaceManager：创建隔离工作区
            workspace_path = self.workspace_manager.create_workspace(
                context["codebase"]["repo_path"]
            )

            # 3. 应用补丁
            for patch in patches:
                patcher = Patcher()
                patcher.apply(patch)

            # 4. SDET：生成测试
            sdet_result = self.sdet.execute({"workspace": workspace_path})
            test_bundle = sdet_result.get("test_bundle")

            # 5. Runner：执行测试
            runner_result = self.runner.run_commands(
                test_bundle.runner_commands,
                workspace_path
            )

            # 6. 判断结果
            if runner_result.passed:
                print(f"\n✅ 第 {iteration} 次迭代：测试通过！")
                return {
                    "success": True,
                    "iterations": iteration,
                    "final_code": patches,
                    "test_results": runner_result,
                    "failure_history": [],
                }

            # 7. 失败：进行诊断
            print(f"\n❌ 第 {iteration} 次迭代：测试失败")

            triage_result = self.triage.execute({
                "sandbox_result": runner_result,
                "code_changes": str(patches),
                "previous_failures": getattr(self, '_failure_history', []),
            })

            failure_analysis = triage_result["failure_analysis"]
            self._failure_history = self._failure_history + [failure_analysis]

            print(f"  错误类型: {failure_analysis.error_type}")
            print(f"  根本原因: {failure_analysis.root_cause}")
            print(f"  修复建议: {failure_analysis.suggestion}")

            # 8. RetryManager：决策是否继续
            if not self.retry_manager.should_continue(failure_analysis):
                print(f"\n⚠️ 已达到重试限制或错误无法恢复")
                return {
                    "success": False,
                    "iterations": iteration,
                    "final_code": patches,
                    "test_results": runner_result,
                    "failure_history": self._failure_history,
                    "final_failure": failure_analysis,
                }

            # 9. 反馈给 CodeGen，准备下一轮迭代
            feedback_for_codegen = {
                "error_type": failure_analysis.error_type,
                "error_message": failure_analysis.error_message,
                "suggestion": failure_analysis.suggestion,
                "failed_code": failure_analysis.code_snippet,
            }

            self.retry_manager.record_failure(failure_analysis)
            print(f"\n准备第 {iteration+1} 次迭代...")
```

## 5. 关键设计考量

| 问题               | 解决方案                                |
| ------------------ | --------------------------------------- |
| **无限循环**       | RetryManager 限制重试次数 + 循环检测    |
| **错误分析准确性** | 多种启发式规则 + LLM 辅助分析           |
| **修复建议质量**   | 将错误映射到 CodeGen 的 prompt 指导     |
| **工作区隔离**     | WorkspaceManager 每次迭代创建独立工作区 |
| **失败历史追踪**   | FailureAnalysis 记录完整上下文          |
| **可观测性**       | 每次迭代打印详细日志 + 生成报告         |
