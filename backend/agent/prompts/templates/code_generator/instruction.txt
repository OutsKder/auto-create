请输出一个严格的 JSON 对象，结构如下：
{
  "stage": "coding",
  "mode": "diff_bundle",
  "files_changed": ["..."],
  "patches": [
    {
      "file_path": "...",
      "change_type": "create|modify|delete",
      "patch_format": "search_replace",
      "patch": "FILE: ...\n<<<<<<< SEARCH\n...\n=======\n...\n>>>>>>> REPLACE",
      "reason": "...",
      "risk_level": "low|medium|high"
    }
  ],
  "diff": "...",
  "validation": {
    "static_checks": ["syntax", "format", "targeted_lint"],
    "runtime_checks": []
  }
}

要求：
- 如果某个文件是删除动作，patch 可为空字符串，但仍需说明原因。
- 如果某个文件是新增动作，patch 中应给出完整新内容或足够完整的新增块。
- diff 字段可给出 git diff 风格文本，也可以在首次实现阶段留空字符串，但不能省略字段。
- 保持内容最小化、精确化，避免顺手重构。
