# 需求交付流程引擎 MVP 实现方案

## 1. 实现目标

本次实现以“可立即试用、可完整演示、后续易扩展”为目标，先落地一个本地可运行的 MVP 版本，覆盖以下核心能力：

- 新建需求并查看需求列表
- 按预设 Pipeline 串联需求分析、方案、编码、测试、评审、交付
- 通过模拟 Agent 自动生成各阶段产物
- 在关键节点支持人工审核、驳回、编辑和重跑
- 对整个流程进行状态展示、日志追踪和交付归档

本版本优先证明“流程引擎 + 多阶段 Agent + 人机协同”的产品闭环，不追求企业级集成能力。

---

## 2. 为什么这样实现

当前工作区为空，且本地环境未检测到 `node`。为了让产品可以尽快试用，本次采用：

- **纯前端零依赖实现**
- **浏览器直接打开即可运行**
- **本地 `localStorage` 持久化数据**

这样做有几个好处：

- 无需安装依赖，最快落地
- 演示环境要求最低
- 便于先验证产品交互和流程设计
- 后续可以平滑迁移到 React / Vue + 后端服务架构

## 5. 产品结构

### 5.1 页面结构

采用单页应用布局，分为三块区域：

1. **左侧需求列表**
   - 展示所有需求
   - 支持查看当前状态、优先级、负责人
   - 支持切换不同需求

2. **中间主工作区**
   - 展示当前需求的基本信息
   - 展示 Pipeline 节点卡片
   - 展示阶段产物
   - 提供执行、审批、驳回、重跑等操作

3. **右侧辅助信息区**
   - 执行日志
   - 风险提示
   - 当前交付总结

### 5.2 主要交互路径

1. 用户填写需求表单并创建需求
2. 系统自动触发“需求分析”
3. 系统自动生成“方案草案”
4. 用户在方案节点审核通过或驳回
5. 审核通过后，系统继续自动执行编码、测试、评审
6. 最终生成交付总结
7. 用户可在任意节点查看内容、编辑备注、重跑流程

---

## 6. 流程设计

### 6.1 节点定义

本次流程固定为：

`需求录入 -> 需求分析 -> 方案设计 -> 编码实现 -> 测试验证 -> 评审确认 -> 交付归档`

### 6.2 节点状态

- `pending`：未开始
- `running`：执行中
- `waiting_review`：等待人工审核
- `approved`：已审核通过
- `rejected`：已驳回
- `completed`：已完成

### 6.3 人工校验节点

本版本将以下节点设为人工审核节点：

- `方案设计`
- `评审确认`

原因：

- 方案是需求交付中最关键的决策点
- 评审是交付前质量把关的关键点

## 8. Agent 设计

### 8.1 实现原则

由于本版本不接真实模型，因此使用“模板生成 + 规则拼装”的方式模拟 Agent 输出，但保持结构化产物形式，便于以后替换成真实 LLM。

### 8.2 Agent 列表

- 需求分析 Agent
  - 输出：需求摘要、澄清问题、验收目标
- 方案设计 Agent
  - 输出：模块拆解、技术方案、风险点、里程碑
- 编码 Agent
  - 输出：实现思路、任务拆分、示例代码骨架
- 测试 Agent
  - 输出：测试用例、验收清单、回归检查点
- 评审 Agent
  - 输出：评审摘要、问题列表、结论建议
- 交付 Agent
  - 输出：交付总结、上线建议、后续迭代建议

### 8.3 未来演进方式

后续可以将每个 Agent 的生成逻辑替换为：

- 调用大模型 API
- 接入真实代码仓库和测试平台
- 引入结构化 Prompt 和工具调用

- 操作按钮

### 9.3 人工审核

对需审核节点提供：

- 通过
- 驳回
- 重跑
- 编辑备注

### 9.4 日志追踪

记录以下关键事件：

- 创建需求
- 节点启动
- 节点完成
- 节点待审核
- 审核通过 / 驳回
- 节点重跑

## 10. 可用性设计

### 10.1 设计目标

- 页面一眼能看懂当前流程走到哪一步
- 所有核心操作按钮明确可见
- 输出内容适合比赛演示，尽量结构化、可阅读
- 空状态、示例需求、提示文案清晰

### 10.2 具体策略

- 初始化内置一条示例需求，降低首次体验门槛
- 使用颜色区分不同阶段状态
- 所有操作都给出即时反馈
- 表单尽量简单，避免过多字段影响试用

## 11. 开发拆解

- 处理等待审核节点

### 11.4 第四步：Agent 输出

## 12. 验收标准

本版本完成后，应满足以下标准：

- 浏览器打开 `index.html` 即可使用
- 可以创建新需求并自动生成流程
- 至少能完整跑通 1 条需求的全链路
- 方案节点和评审节点支持人工审核
- 日志和交付总结能正常展示
- 页面在无依赖环境下稳定运行
